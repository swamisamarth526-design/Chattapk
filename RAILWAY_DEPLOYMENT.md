# ChatX - Railway Deployment Guide

This guide explains how to deploy ChatX on Railway as a single Node.js/Express service with a separate MongoDB service.

## Project Structure

```
ChatX/
├── package.json              # Root package.json (coordinates builds)
├── client/                   # React frontend (Vite)
│   ├── package.json
│   ├── vite.config.js
│   ├── src/
│   └── dist/                 # Built React app (generated by build)
├── server/                   # Express backend + Socket.IO
│   ├── package.json
│   ├── index.js              # Main server entry
│   ├── src/
│   ├── .env.example
│   └── node_modules/
└── README.md
```

## Build & Deployment Flow

### 1. Local Development

```bash
# Install all dependencies
npm run install-all

# Start client (React dev server)
npm run dev:client
# Runs on http://localhost:5173

# Start server (Node.js with Socket.IO)
npm run dev:server
# Runs on http://localhost:5000
# API proxy: http://localhost:5173/api → http://localhost:5000/api
```

### 2. Railway Build Process

Railway automatically executes on deployment:

```bash
# Root install (installs root deps if any)
npm install

# Root build
npm run build

# This runs:
# 1. cd client && npm install && npm run build
#    → Creates client/dist with optimized React bundle
# 2. cd server && npm install
#    → Installs server dependencies (no build needed)
```

### 3. Railway Start Process

```bash
# Executes start script from root package.json
npm start

# This runs:
# 1. cd server && npm start
#    → Starts Node.js server on process.env.PORT
#    → Express serves client/dist/ as static files
#    → Socket.IO ready on same port
```

### 4. File Serving at Runtime

When client visits `/`:
```
Request → Express (port 5000)
  ├─ Check if /api route? → Handle API
  ├─ Check if static file (js, css, img)? → Serve from client/dist
  └─ Otherwise → Serve index.html (SPA routing)
```

## Setup on Railway

### Step 1: Create Railway Account
- Go to https://railway.app
- Sign in with GitHub/GitLab/GitBucket

### Step 2: Create New Project
- Click "New Project"
- Select "Deploy from GitHub"
- Select your ChatX repository
- Authorize Railway with GitHub

### Step 3: Configure Environment Variables

In Railway dashboard, set these environment variables:

**For Express Server:**
```env
NODE_ENV=production
JWT_SECRET=<generate-strong-128-char-secret>
MONGODB_URI=<provided-by-MongoDB-service>
PORT=5000
```

**Generate JWT_SECRET:**
```bash
# Run locally
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### Step 4: Add MongoDB Service
1. In Railway project, click "Add"
2. Select "Database"
3. Choose "MongoDB"
4. Railway auto-adds `MONGODB_URI` env variable

### Step 5: Verify Build Settings

Railway should auto-detect:
- **Build command:** `npm run build`
- **Start command:** `npm start`
- **Root directory:** `/` (if not, set to root)

If not auto-detected, set manually:
1. Click on your service
2. Go to "Settings"
3. Set Deploy section:
   - Build command: `npm run build`
   - Start command: `npm start`
   - Root directory: (leave blank for root)

### Step 6: Deploy

- Push code to GitHub
- Railway auto-deploys on push
- Check logs for build/start output

## Common Issues & Solutions

### Issue: Build fails with "client/dist not found"
**Solution:** Ensure root package.json has correct build script:
```json
{
  "scripts": {
    "build": "npm run build --prefix client && npm run build --prefix server"
  }
}
```

### Issue: 404 on page refresh
**Solution:** Express must serve React app for all non-API routes. Check `server/src/config/app.js`:
```javascript
// Catch-all for SPA
app.get('*', (req, res) => {
  res.sendFile(path.join(clientBuildPath, 'index.html'));
});
```

### Issue: Socket.IO connection fails in production
**Solution:** CORS is auto-handled when frontend served from same origin. Check Socket.IO config in `server/src\sockets/index.js`:
```javascript
const io = socketIo(server, {
  cors: {
    origin: allowedOrigins,  // '*' in production
    methods: ['GET', 'POST'],
    credentials: true,
  },
});
```

### Issue: Static files not served (CSS/JS 404)
**Cause:** Express static middleware not configured
**Solution:** Check `app.js` line ~60:
```javascript
if (!isDev) {
  const clientBuildPath = path.join(__dirname, '../../client/dist');
  app.use(express.static(clientBuildPath));
}
```

### Issue: Cannot read property 'MONGODB_URI' undefined
**Solution:** Railway MongoDB hasn't booted yet or connection string not set. Check:
1. MongoDB service exists in Railway project
2. MONGODB_URI env var shown in dashboard
3. Server logs show connection success

## Performance Notes

### Frontend Optimization
- React bundle is minified and chunked
- Code splitting: vendor chunks separated
- CSS and JS minified
- Images should be optimized before deploy

### Backend Optimization
- Socket.IO in same server (no extra latency)
- Static files served efficiently
- API responses should be fast
- MongoDB indexes help query performance

### Scaling (if needed)
For multiple instances, add Redis:
1. Railway → Add → Database → Redis
2. Update Socket.IO config to use Redis adapter
3. See `SOCKETIO_INTEGRATION.md` for details

## Rollback & Recovery

### Rollback to Previous Deployment
1. Railway → Deployments
2. Click previous version
3. Click "Redeploy"

### Clear Cache (force rebuild)
1. In Railway settings, no direct cache clear button
2. Force rebuild by pushing a commit

## File Mappings

| Local Path | Purpose | Served From |
|---|---|---|
| `client/src/` | React source | Transpiled at build |
| `client/dist/` | Built React (static) | Express `/` route |
| `server/src/` | Backend code | Node.js runtime |
| `server/src/config/app.js` | Express config | Handles routes, CORS, static files |
| `server/src/sockets/` | Socket.IO handlers | WebSocket runtime |

## Summary

✅ **Deployment Ready When:**
- Root `package.json` has build and start scripts
- `client/dist/` will be generated by build
- `server/index.js` listens on `process.env.PORT`
- `MONGODB_URI` will be provided by Railway
- `JWT_SECRET` set in Railway environment
- CORS handles production same-origin
- Socket.IO configured for production

✅ **No Docker Needed** - Railway Node.js buildpack handles everything

✅ **One Service** - Express serves both API and frontend

✅ **Separate Database** - MongoDB as Railway add-on service
